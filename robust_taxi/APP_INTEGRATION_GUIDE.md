# 雙螢幕互動與緊急廣播系統 - App 端開發指南 (V2)

本文件說明如何對接後端系統，實現 **兩台獨立設備（螢幕 A 與 螢幕 B）** 的協作控制與緊急插播功能。

---

## 1. 系統架構：雙機協作

本系統由 **後端 Server** 同時控制兩台安裝了 App 的設備。
App 需具備 **「角色設定 (Role Configuration)」**，在啟動時決定自己是「螢幕 A」還是「螢幕 B」。

- **Server (中控)**：負責廣播「系統狀態 (State)」，不個別發送指令。
- **App (用戶端)**：兩台 App 連接同一個 Server，收到狀態後，依據自己的角色執行對應動作。

### 角色定義
請在 App 的設定檔或啟動參數中加入 `DEVICE_ROLE` 設定：

| 角色代號 | 設備名稱 | 職責 | 平時行為 | 警報時行為 |
| :--- | :--- | :--- | :--- | :--- |
| **SCREEN_A** | 全功能廣告屏 | 多媒體播放 | 播放廣告輪播 | **疊加跑馬燈** (不中斷廣告) |
| **SCREEN_B** | 互動情境屏 | 資訊與互動 | 播放廣告 + **QR Code 統計** | **中斷廣告**，改播地震影片 |

---

## 2. WebSocket 連接與初始化

- **連線 URL**: `ws://<SERVER_IP>:8080`
- **Namespace**: `/`

### Step 1: 連線與註冊
連線後，請發送 `register` 事件。建議在註冊資訊中帶上角色資訊（供後台識別，雖非必要但建議）：
```json
// 發送 register
{
    "device_id": "taxi-01-screen-A",
    "device_type": "SCREEN_A" // 或 SCREEN_B
}
```

### Step 2: 接收初始狀態
註冊成功後，Server 會自動推送 `system_state_update`。App 應立即根據此狀態更新畫面（例如：若 Server 正處於警報狀態，App 一開機就該進入警報模式）。

---

## 3. 核心事件監聽 (Event Listeners)

兩台 App 監聽 **相同的事件**，但根據角色 **執行不同的邏輯**。

### A. 系統狀態更新 (`system_state_update`)

當警報觸發/解除，或跑馬燈更新時收到。

**Payload**:
```json
{
    "is_alarm_active": true,                     // true=警報模式, false=正常模式
    "marquee_text": "地震速報：請保持冷靜...",      // 跑馬燈文字
    "emergency_video": "earthquake_alert.mp4",   // 警報影片檔名 (B屏用)
    "timestamp": "2026-01-12T14:30:00"
}
```

**處理邏輯對照表**:

| 收到資料 | 螢幕 A (SCREEN_A) 的反應 | 螢幕 B (SCREEN_B) 的反應 |
| :--- | :--- | :--- |
| **`is_alarm_active: false`**<br>(正常模式) | 1. 繼續播放廣告<br>2. **隱藏**跑馬燈 | 1. 繼續播放廣告<br>2. **顯示** QR Code 統計層 |
| **`is_alarm_active: true`**<br>(警報模式) | 1. 繼續播放廣告 (不中斷)<br>2. **顯示**紅色跑馬燈 (`marquee_text`) | 1. **立即暫停/停止**當前廣告<br>2. **隱藏** QR Code 統計層<br>3. **強制播放** `emergency_video` |

> **⚠️ 重要**：`emergency_video` 建議 App 在閒置時預先下載至本地，以確保警報觸發時能達成 **0 秒延遲切換**。

---

### B. QR 統計更新 (`stats_update`)

當有人掃描 QR Code 時收到。

**Payload**:
```json
{
    "qr_scan_count": 105,
    "timestamp": "..."
}
```

**處理邏輯**:
- **螢幕 A**: (忽略此事件)
- **螢幕 B**: 更新畫面角落的「掃描人數」數字。

---

## 4. 自我測試清單 (App 工程師用)

請使用我們提供的網頁中控台進行測試：`http://<SERVER_IP>:8080/control`

1. **[B屏] QR 數據測試**:
   - 用手機掃描 `http://<SERVER_IP>:8080/qr-scan`。
   - 確認 **螢幕 B** 的數字即時增加。
   - 確認 **螢幕 A** 完全無反應（正確）。

2. **[雙屏] 警報觸發測試**:
   - 在中控台點擊「🔴 觸發警報」。
   - 確認 **螢幕 A** 出現跑馬燈。
   - 確認 **螢幕 B** 畫面全被地震影片覆蓋。

3. **[雙屏] 警報解除測試**:
   - 在中控台點擊「🟢 恢復正常」。
   - 確認 **螢幕 A** 跑馬燈消失。
   - 確認 **螢幕 B** 切回原本的廣告輪播。
